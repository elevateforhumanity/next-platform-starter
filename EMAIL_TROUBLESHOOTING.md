## Issue: Emails Sent to info@elevateforhumanity.org Going to Wrong Place

### Possible Causes

1. **MX Records Point to Wrong Service**
   - Check if MX records are pointing to Google Workspace, Microsoft 365, or another provider
   - If using SendGrid, MX should point to SendGrid's inbound parse server

2. **SendGrid Domain Verification**
   - Domain must be verified in SendGrid settings
   - Check: Settings → Sender Authentication → Domain Authentication

3. **Email Forwarding Rules**
   - Check if info@elevateforhumanity.org has forwarding rules set up
   - Check email provider's routing settings

## Diagnostic Steps

### Step 1: Check Current MX Records

```bash
nslookup -type=MX elevateforhumanity.org
```

Expected for SendGrid:
```
mx.sendgrid.net
```

Expected for Google Workspace:
```
aspmx.l.google.com
alt1.aspmx.l.google.com
```

### Step 2: Check SPF Record

```bash
nslookup -type=TXT elevateforhumanity.org
```

Should include:
```
v=spf1 include:sendgrid.net ~all
```

### Step 3: Check DKIM Record

In SendGrid: Settings → Sender Authentication → Domain Authentication → DKIM

## Common Solutions

### Solution 1: Verify Domain in SendGrid

1. Log into SendGrid: https://app.sendgrid.com
2. Go to Settings → Sender Authentication
3. Click "Authenticate Your Domain"
4. Enter: `elevateforhumanity.org`
5. Add the DNS records shown
6. Wait for verification (can take up to 48 hours)

### Solution 2: Use Different From Address

If domain verification is pending, use an email from a verified domain:

```javascript
// In send-conference-submission.ts
from: {
  email: 'noreply@elevateforhumanity.org', // or
  email: 'elevate4humanityedu@gmail.com',
  name: 'Elevate for Humanity',
}
```

### Solution 3: Check Email Provider Forwarding

If using Google Workspace:
1. Go to admin.google.com
2. Apps → Google Workspace → Gmail → End User Access
3. Check for forwarding rules
4. Remove any rules forwarding to other addresses
